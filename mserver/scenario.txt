----------------------------------------------------------------------------------------------------

----------------------------------------------------------------------------------------------------
recv
----------------------------------------------------------------------------------------------------
<recv message="msg-name" [optional] [timeout=NUM]>
	Header: expectation // should this be supported on just use expect command?
</recv>

message must be the first parameter

----------------------------------------------------------------------------------------------------
expect
----------------------------------------------------------------------------------------------------
<expect>
    boolean-expression
</expect>

----------------------------------------------------------------------------------------------------
send
----------------------------------------------------------------------------------------------------
<send [last="<last-descriptor"]>
	Literal stuff containing place-holders
</send>

----------------------------------------------------------------------------------------------------
pause
----------------------------------------------------------------------------------------------------
<pause milliseconds=1000/>


Can be exactly like in sipp


====================================================================================================
place-holders/keywords
====================================================================================================
[last_<Header>] 			like sipp
[last_<Header>_value] 		not containing the header
[(remote/loacl)_(ip/port)]	like sipp

[other-name] : if other-name is not a known keyword, it is a place holder for a parameterized scenario
